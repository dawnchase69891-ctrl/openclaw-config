## 执行摘要

P1 模块包含三个核心功能：
1. **GitHub Actions 工作流** - 每 5 分钟自动同步配置
2. **单元测试完善** - 覆盖率目标 70%+
3. **飞书通知集成** - 告警、任务通知、错误报告

所有功能已实现并通过测试验证。

## 一、GitHub Actions 工作流

### 1.1 文件位置

```
.github/workflows/sync-config.yml
```

### 1.2 功能特性

- ✅ 每 5 分钟自动执行 (`*/5 * * * *`)
- ✅ 支持手动触发 (`workflow_dispatch`)
- ✅ 配置验证步骤
- ✅ 失败回滚机制 (通过备份)
- ✅ Git 自动提交和推送
- ✅ 成功/失败状态输出

### 1.3 工作流程

```
1. Checkout 代码
   ↓
2. 设置 Python 环境
   ↓
3. 安装依赖 (requests, jsonschema)
   ↓
4. 从飞书 Bitable 读取配置
   ↓
5. 验证配置格式 (JSON Schema)
   ↓
6. Git 提交变更
   ↓
7. 推送至仓库
   ↓
8. 输出结果
```

### 1.4 安全配置

需要在 GitHub 仓库设置中添加以下 Secrets:

| Secret 名称 | 说明 | 获取方式 |
|------------|------|----------|
| `FEISHU_APP_ID` | 飞书应用 ID | 飞书开发者后台 |
| `FEISHU_APP_SECRET` | 飞书应用密钥 | 飞书开发者后台 |
| `FEISHU_BITABLE_TOKEN` | 多维表格 Token | 飞书多维表格 URL |
| `GITHUB_TOKEN` | GitHub Token | 自动提供 |

### 1.5 手动触发

在 GitHub 仓库页面：
1. 进入 **Actions** 标签
2. 选择 **Sync Agent Config** 工作流
3. 点击 **Run workflow**
4. 选择分支 (默认 `main`)
5. 点击 **Run workflow** 按钮

## 四、飞书通知集成

### 4.1 文件位置

```
scripts/feishu_notifier.py
```

### 4.2 功能特性

- ✅ 支持交互式消息卡片
- ✅ 支持纯文本消息
- ✅ 消息去重 (5 分钟窗口)
- ✅ 支持@用户
- ✅ 多种通知类型 (告警/任务完成/任务失败/错误报告/信息)
- ✅ 可选的签名验证
- ✅ 全局单例模式

### 4.3 配置方式

#### 4.3.1 获取 Webhook URL

1. 打开飞书群
2. 点击右上角设置图标
3. 选择 **群机器人**
4. 点击 **添加机器人**
5. 选择 **自定义机器人**
6. 复制 Webhook 地址

#### 4.3.2 设置环境变量

```bash
export FEISHU_WEBHOOK_URL="https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
export FEISHU_WEBHOOK_SECRET="xxx"  # 可选，如果启用了签名验证
```

### 4.4 使用方式

#### 4.4.1 命令行使用

```bash
# 发送告警
python3 scripts/feishu_notifier.py alert "配置同步失败" "请检查日志"

# 发送任务完成通知
python3 scripts/feishu_notifier.py task-complete TASK-001 ClawBuilder "实现登录功能"

# 发送错误报告
python3 scripts/feishu_notifier.py error "同步脚本" "ConnectionError" "无法连接到飞书 API"

# 发送测试消息
python3 scripts/feishu_notifier.py test
```

#### 4.4.2 Python 代码使用

```python
from feishu_notifier import FeishuNotifier, NotificationConfig

# 方式 1: 使用环境变量
notifier = FeishuNotifier()

# 方式 2: 直接传入配置
config = NotificationConfig(
    webhook_url="https://open.feishu.cn/open-apis/bot/v2/hook/xxx",
    secret="xxx",  # 可选
    mention_users=["ou_xxx"]  # 可选，要@的用户
)
notifier = FeishuNotifier(config=config)

# 发送告警
notifier.send_alert(
    alert_title="配置同步失败",
    alert_content="请检查日志"
)

# 发送任务完成通知
notifier.send_task_complete(
    task_id="TASK-001",
    assignee="ClawBuilder",
    task_description="实现登录功能",
    duration="2h 30m"
)

# 发送错误报告
notifier.send_error_report(
    module="同步脚本",
    error_type="ConnectionError",
    error_message="无法连接到飞书 API",
    stack_trace="Traceback..."
)

# 发送普通信息
notifier.send_info(
    title="系统通知",
    content="配置已更新"
)
```

#### 4.4.3 便捷函数

```python
from feishu_notifier import send_alert, send_task_complete, send_error_report

# 使用全局通知器 (需要先配置环境变量)
send_alert("告警标题", "告警内容")
send_task_complete("TASK-001", "ClawBuilder", "任务描述")
send_error_report("模块名", "错误类型", "错误信息")
```

### 4.5 集成示例

#### 4.5.1 在配置同步脚本中集成

```python
from feishu_notifier import send_alert, send_task_complete

def sync():
    try:
        # 执行同步...
        send_task_complete(
            task_id="SYNC-001",
            assignee="System",
            task_description="配置同步",
            duration="30s"
        )
    except Exception as e:
        send_alert(
            alert_title="配置同步失败",
            alert_content=str(e)
        )
```

```python

    try:
        # 发送消息...
    except AgentOfflineError as e:
        send_error_report(
            module="协作工具",
            error_type="AgentOfflineError",
            error_message=str(e)
```

## 八、故障排查

### 8.1 配置同步失败

**问题**: `python3 sync_feishu_to_openclaw.py` 执行失败

**排查步骤**:
1. 检查环境变量是否设置
   ```bash
   echo $FEISHU_APP_ID
   echo $FEISHU_APP_SECRET
   ```
2. 检查飞书应用权限
3. 检查多维表格 Token 是否正确
4. 查看错误日志

**解决方案**:
```bash
# 重新设置环境变量
export FEISHU_APP_ID=cli_xxx
export FEISHU_APP_SECRET=xxx
export FEISHU_BITABLE_TOKEN=xxx

# 重新执行
python3 scripts/sync_feishu_to_openclaw.py
```

### 8.2 飞书通知发送失败

**问题**: 通知发送失败或收不到消息

**排查步骤**:
1. 检查 Webhook URL 是否正确
2. 检查网络连接
3. 检查飞书群机器人是否启用
4. 查看返回错误信息

**解决方案**:
```bash
# 测试通知
python3 scripts/feishu_notifier.py test

# 检查环境变量
echo $FEISHU_WEBHOOK_URL
```

### 8.3 测试失败

**问题**: `pytest` 运行失败

**排查步骤**:
1. 检查依赖是否安装
   ```bash
   pip install -r requirements-p1.txt
   ```
2. 检查 pytest 版本
   ```bash
   pytest --version
   ```
3. 查看详细错误信息

**解决方案**:
```bash
# 重新安装依赖
pip install --upgrade -r requirements-p1.txt

# 清除 pytest 缓存
rm -rf .pytest_cache __pycache__

# 重新运行测试
pytest tests/ -v
```

### 8.4 GitHub Actions 不执行

**问题**: 工作流没有按预期执行

**排查步骤**:
1. 检查 `.github/workflows/sync-config.yml` 语法
2. 检查 Secrets 是否配置
3. 查看 Actions 日志
4. 检查 cron 表达式

**解决方案**:
```bash
# 验证 YAML 语法
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/sync-config.yml'))"

# 手动触发工作流
# 在 GitHub 页面 -> Actions -> Sync Agent Config -> Run workflow
```

## 九、最佳实践

### 9.1 配置管理

1. **使用环境变量**: 敏感信息不要硬编码
2. **定期备份**: 配置同步会自动备份，但建议定期手动备份
3. **版本控制**: 所有配置文件纳入 Git 管理
4. **权限分离**: 飞书应用权限最小化原则

### 9.2 通知策略

1. **避免骚扰**: 使用消息去重功能
2. **分级通知**: 
   - 告警 → @相关人员
   - 任务完成 → 普通通知
   - 错误报告 → 详细堆栈
3. **定时汇总**: 非紧急错误定时汇总发送

### 9.3 测试规范

1. **测试覆盖**: 新功能必须包含测试
2. **Mock 外部依赖**: 避免测试依赖网络
3. **CI 集成**: 在 GitHub Actions 中运行测试
4. **覆盖率要求**: 核心模块 >= 80%
