## 🎯 原则

### 核心原则

1. **专业人干专业事** - 每个角色做自己专业的事
2. **结果导向** - 关注产出质量，不只是完成任务
3. **先验证后修改** - 配置文件必须先测试再变更
4. **最小权限** - 只修改必要的内容
5. **可回滚** - 每次变更都要有回滚方案

---

## 📊 配置分级管理

### P0 - 核心配置 (需 CEO 审批)

| 配置文件 | 影响范围 | 审批流程 |
|---------|---------|---------|
| `openclaw.json` | 全局 | 骐骥 (CEO) → Rex → ClawOps |
| `agents.list` | 全局 | 骐骥 (CEO) → Rex |
| `plugins.allow` | 全局 | 骐骥 (CEO) → ClawBreaker |

### P1 - 重要配置 (需 Rex 审批)

| 配置文件 | 影响范围 | 审批流程 |
|---------|---------|---------|
| `agents.defaults` | 全局 | Rex → ClawBreaker |
| `tools.profile` | 全局 | Rex → ClawGuard |
| `channels.*` | 全局 | Rex → ClawOps |

### P2 - 一般配置 (角色负责人审批)

| 配置文件 | 影响范围 | 审批流程 |
|---------|---------|---------|
| SKILL 文件 | 单角色 | 角色负责人 → ClawCoordinator |
| 文档文件 | 单角色 | 角色负责人 |

## 🔄 配置变更流程

### 标准流程

```
1. 变更申请
   └─→ 申请人填写《配置变更申请表》
   
2. 影响评估 ⚠️
   ├─→ Rex: 业务影响
   ├─→ ClawBreaker: 技术影响
   └─→ ClawGuard: 风险评估
   
3. 测试验证 ⚠️ (必须执行)
   ├─→ 语法验证
   ├─→ 测试环境部署
   └─→ 功能测试
   
4. 审批
   └─→ 根据配置级别审批
   
5. 生产部署
   ├─→ 备份原配置
   ├─→ 灰度发布
   └─→ 监控验证
   
6. 复盘
   └─→ 变更记录
   └─→ 经验教训
```

## 🛡️ 风险控制

### 变更风险等级

| 等级 | 定义 | 示例 | 审批要求 |
|------|------|------|---------|
| 🔴 高风险 | 影响全局，可能中断服务 | openclaw.json, plugins | CEO + Rex + ClawBreaker |
| 🟠 中风险 | 影响部分功能 | agents.defaults, tools | Rex + ClawGuard |
| 🟡 低风险 | 影响单个角色 | SKILL 文件，文档 | 角色负责人 |

### 回滚方案

每次变更必须准备：

1. **备份原配置**
   ```bash
   cp openclaw.json openclaw.json.backup.$(date +%Y%m%d_%H%M%S)
   ```

2. **回滚命令**
   ```bash
   # 快速回滚脚本
   ./rollback.sh <timestamp>
   ```

3. **验证回滚**
   ```bash
   openclaw gateway status
   openclaw agents list
   ```

## 🎯 角色职责

### 骐骥 (CEO) 🐎
- P0 配置最终审批
- 重大变更决策
- 资源分配审批

### Rex (产品指挥官) 🎯
- P1 配置审批
- 业务影响评估
- 变更优先级决策

### ClawBreaker (系统架构师) 🔧
- 技术架构评审
- 配置语法验证
- 技术风险评估

### ClawGuard (测试工程师) 🛡️
- 配置测试验证
- 质量门禁检查
- 回归测试

### ClawOps (运维工程师) 🛠️
- 生产部署执行
- 监控配置
- 回滚执行

### ClawCoordinator (项目协调员) 📋
- 变更流程跟踪
- 变更记录管理
- 经验教训沉淀

- 文档归档
- 合规检查
- 用户通知

## ⚠️ 禁止行为

### 绝对禁止

- ❌ 未测试直接修改生产配置
- ❌ 跳过审批流程
- ❌ 不备份直接修改
- ❌ 无回滚方案执行变更
- ❌ 非工作时间执行高风险变更

### 不推荐行为

- ⚠️ 多人同时修改同一配置
- ⚠️ 直接编辑生产环境配置
- ⚠️ 变更不记录
- ⚠️ 失败不复盘
