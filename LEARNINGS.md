## 附录：学习触发机制

### 自动化触发条件

**评估触发**:
- 条件1: `github-learning-candidates.md`中有≥3个"待评估"项目
- 条件2: 距离上一次评估≥7天
- 触发动作: 自动提醒ClawBreaker完成评估任务
- Cron: 每周一10:00

**学习总结触发**:
- 条件: 每周日18:00
- 触发动作: 生成本周学习总结，更新LEARNINGS.md
- Cron: 每周日18:00

**KPI报告触发**:
- 条件: 每周日19:00
- 触发动作: 生成KPI周报，发送到ClawSquad协作群
- Cron: 每周日19:00

**学习推广触发**:
- 条件: 每周日20:00
- 触发动作: 扫描高优先级学习，推广到AGENTS.md/TOOLS.md/SOUL.md
- Cron: 每周日20:00

### 学习生命周期

```
【发现阶段】ClawOps每日扫描GitHub
    ↓
【评估阶段】PM + Dev评估技术价值和业务相关性
    ↓
【学习阶段】Dev深度学习项目源码和架构
    ↓
【存储阶段】存储到learning-outputs/{project-name}/
    ↓
【应用阶段】集成到现有系统或创建新能力
    ↓
【追踪阶段】记录应用效果和ROI
```

### 学习KPI计算

```python
# KPI计算公式
评估率 = 完成评估数 / 发现候选数
学习率 = 完成学习数 / 完成评估数
应用率 = 成功应用数 / 完成学习数
转化率(评估) = 评估率 * 学习率
转化率(应用) = 应用率
综合得分 = (评估率*30 + 学习率*40 + 应用率*30) / 100
```

**目标**:
- 评估率 ≥80%
- 学习率 ≥60%
- 应用率 ≥30%
- 转化率(评估) ≥48%
- 转化率(应用) ≥30%
- 综合得分 ≥50分
